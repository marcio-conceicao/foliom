Wire the notify-debouncer-full filesystem watcher into the axum serve command.     Internal structure of spawn_watcher (per D-40-01, D-40-03, RESEARCH patterns):
    1. Create std::sync::mpsc::channel::<DebounceEventResult>().
    2. Call new_debouncer(Duration::from_millis(debounce_ms), None, mpsc_tx) — mpsc::Sender implements DebounceEventHandler.
    3. Call debouncer.watcher().watch(&root, RecursiveMode::Recursive).
    4. Call debouncer.cache().add_root(&root, RecursiveMode::Recursive) for rename tracking.
    5. std::thread::spawn with _debouncer moved in (keeps alive) and a DashMap<PathBuf,()> dirty set.
    6. In the event loop (for result in &mpsc_rx):
       - Ok(events): check events.iter().any(|e| e.event.need_rescan()); if true → call do_full_reindex_blocking(&root, &db) → broadcast_tx.send(WatcherEvent::IndexReset).ok(); continue.
       - Otherwise: for each event path, filter .md extension only (do NOT filter by EventKind — macOS/Windows may emit EventKind::Any). Read file bytes, compute blake3 hash, call self_writes.take_if_present(&hash). If suppressed skip; else insert path into dirty.
       - Err(errors): log each error with tracing::warn; call do_full_reindex_blocking; broadcast IndexReset; also re-arm: call debouncer.watcher().watch(&root, RecursiveMode::Recursive) to recover from Windows ReadDirectoryChangesW unwatch (SNC-04 per D-40-05 and Q5 answer). Continue.
    7. Coalescing drain: after processing each batch, if last_drain.elapsed() >= 300ms AND dirty not empty → collect paths, clear dirty, call reindex_dirty_files_blocking(&paths, &root, &db) which returns Vec<(String, String)> of (page_name, file_hash), build WatcherEvent::PagesUpdated, send on broadcast_tx. Reset last_drain.

    Path-traversal guard (SECURITY — load-bearing per RESEARCH security domain): in the dirty path insertion loop, call path.canonicalize().ok() and check canonical.starts_with(&root) before inserting. Paths that escape root are logged at warn level and skipped.

    do_full_reindex_blocking: acquires Mutex<Db>, calls the existing foliom_core::indexer::reindex(IndexMode::Incremental, &root, &db). reindex_dirty_files_blocking: same but per-file (call reindex on only the dirty paths by setting mtime to 0 in memory or by calling a per-file variant if available; fall back to full incremental reindex if per-file variant does not exist — full incremental is always safe since it's hash-gated).

    4. Call spawn_watcher(root.clone(), state.self_writes.clone(), state.watcher_tx.clone(), state.db.clone(), debounce_ms)? after AppState is built and before axum serve loop.
