axum server and Svelte SPA. No new backend logic — the shell calls `serve::run()` in a background

own tiny_http server — NOT a bridge to axum; using it would create a duplicate server).

**Step A — Modify crates/cli/src/cmd/serve/mod.rs (per D-50-02):**

Add to the top of the file (after existing `use` imports):
```
use std::sync::OnceLock;
pub static BOUND_PORT: OnceLock<u16> = OnceLock::new();
```

In the `run()` function, after the line `let url = format!("http://{bound}");` and BEFORE `if args.open {`, insert:
```
let _ = BOUND_PORT.set(bound.port());
```

This publishes the port before `rt.block_on(...)` so the Tauri setup hook's poll loop succeeds
within milliseconds. The `let _ =` ignores a double-set (only relevant if run() is called twice,
which never happens, but suppresses the compiler warning).

Do NOT change the function signature or any other logic.

4. Setup hook logic (in order, per research Pattern 1 + D-50-03):

   **Step 1 — Read stored vault root from tauri-plugin-store:**
   Open the store file `"config.json"` using `app.store("config.json")`. This returns a
   `Result<StoreBuilder, ...>` in tauri-plugin-store 2.x — check the actual API. If the store
   cannot be opened, return the error. Try to get the key `"vault_root"` as a `serde_json::Value`.
   If present and a valid string, convert it to `PathBuf`.

   **Step 2 — Native folder picker on first launch (per D-50-03):**
   If no stored `vault_root`, use `tauri_plugin_dialog::blocking::FileDialogBuilder::new().pick_folder()`.
   NOTE (Research Assumption A1): The exact API method `blocking::FileDialogBuilder::new().pick_folder()`
   is ASSUMED from research. Verify against actual tauri-plugin-dialog 2.7.x API at compile time.
   Alternative if `blocking` module does not exist: use `tauri_plugin_dialog::DialogExt::dialog(app).file().pick_folder_blocking()`.
   If the user cancels (returns `None`), return `Err(anyhow::anyhow!("Nenhuma pasta selecionada — encerrando"))`.
   After picking, persist to store: set key `"vault_root"` with `serde_json::Value::String(chosen.to_string_lossy().into_owned())` and call `.save()`.

   **Step 3 — Spawn axum server in background OS thread (per D-50-01 + research Pitfall 2):**
   ```rust
   let root_clone = root.clone();
   std::thread::spawn(move || {
       if let Err(e) = serve_run(ServeArgs {
           root: root_clone,
           port: 0,   // OS picks free port (per D-50-02 — avoids port conflict)
           open: false,
           full: false,
       }) {
           tracing::error!(error = %e, "servidor axum encerrou com erro");
       }
   });
   ```
   CRITICAL: Use `std::thread::spawn`, NEVER `tauri::async_runtime::spawn`. serve::run() builds
   its own `tokio::runtime::Builder::new_current_thread()` and calls `.block_on()`. Calling it
   inside Tauri's tokio context would cause "Cannot start a runtime from within a runtime" panic.

   **Step 4 — Poll BOUND_PORT OnceLock (per D-50-02 + research Pitfall 3):**
   ```rust
   let port = loop {
       if let Some(p) = BOUND_PORT.get() {
           break *p;
       }
       std::thread::sleep(Duration::from_millis(20));
   };
   ```
   BOUND_PORT is set by serve::run() after `bind_loopback()` returns and BEFORE `rt.block_on()`.
   In practice the port is available within 10–50 ms. Add a safety timeout of 5 s (250 iterations
   × 20 ms) and return an error if exceeded:
   ```rust
   let port = {
       let mut attempts = 0u32;
       loop {
           if let Some(p) = BOUND_PORT.get() { break *p; }
           std::thread::sleep(Duration::from_millis(20));
           attempts += 1;
           if attempts > 250 {
               return Err(anyhow::anyhow!("timeout: servidor axum não iniciou em 5s").into());
           }
       }
   };
   ```

   **Step 5 — Open the Tauri window (per research Pattern 1):**
   ```rust
   let url: tauri::Url = format!("http://127.0.0.1:{port}/")
       .parse()
       .expect("URL inválida");
   WebviewWindowBuilder::new(app, "main", WebviewUrl::External(url))
       .title("Foliom")
       .inner_size(1280.0, 800.0)
       .build()?;
   Ok(())
   ```

5. The file must compile without warnings. Use `#[allow(unused_imports)]` only if genuinely

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing | WebviewUrl::External loading arbitrary URLs | mitigate | capabilities/default.json remote.urls allowlist restricts to http://127.0.0.1:*/** — Tauri rejects other origins |
| T-05-02 | Tampering | config.json vault_root path traversal | mitigate | serve::run() opens Db::open(&root) which validates the path exists; non-existent path returns an error before any file IO |
| T-05-03 | Repudiation | No audit log for vault selection | accept | Single-user local app; no PII, no multi-tenant; vault path is user-chosen |
| T-05-04 | Information Disclosure | axum binds 0.0.0.0 in Tauri build | mitigate | bind_loopback() in serve::run() binds 127.0.0.1 (loopback only) — same as Phase 2; T-02-01 mitigation already in place |
| T-05-05 | Denial of Service | OnceLock poll timeout — app hangs if axum fails to start | mitigate | 5s timeout (250×20ms) returns Err; Tauri shows error and exits cleanly rather than hanging |
| T-05-06 | Elevation of Privilege | Tauri IPC not used | accept | D-50-01 explicitly prohibits Tauri IPC commands; no command surface to escalate through |
| T-05-SC | Tampering | npm/pip/cargo installs (tauri, tauri-plugin-dialog, tauri-plugin-store) | mitigate | All three packages verified as official Tauri org crates in 05-RESEARCH §Package Legitimacy Audit with [OK] slopcheck; no [ASSUMED]/[SUS] packages |
