  <action>
Create `.github/workflows/release.yml`. This is a new file — do NOT modify `ci.yml`.

The workflow must implement exactly what is described in 05-RESEARCH Pattern 5 with the following
structure and rules:

**Trigger:** `push: tags: ["v*"]` only. NOT triggered by push/PR (that is ci.yml's job).

**Concurrency:** `group: release-${{ github.ref }}, cancel-in-progress: true`

**Job: `release`**
- `permissions: contents: write` (required for GitHub Release creation)
- `strategy: fail-fast: false`
- Matrix with two entries:
  - `{ platform: macos-latest, args: "--target universal-apple-darwin", rust_targets: "aarch64-apple-darwin,x86_64-apple-darwin" }`
  - `{ platform: windows-latest, args: "--target x86_64-pc-windows-msvc", rust_targets: "x86_64-pc-windows-msvc" }`
- `runs-on: ${{ matrix.platform }}`

**Steps in order:**

1. `actions/checkout@v4`

2. `actions/setup-node@v4` with `node-version: '20'`, `cache: 'npm'`, `cache-dependency-path: frontend/package-lock.json`

3. Build frontend step (name: "Build frontend"):
   ```yaml
   working-directory: frontend
   run: npm ci && npm run build
   ```

4. `dtolnay/rust-toolchain@stable` with `targets: ${{ matrix.rust_targets }}`

5. `Swatinem/rust-cache@v2` with:
   ```yaml
   workspaces: ". -> target"
   shared-key: phase5-${{ matrix.platform }}
   ```

6. macOS certificate import (name: "Import Apple signing certificate"):
   ```yaml
   if: runner.os == 'macOS' && secrets.APPLE_CERTIFICATE != ''
   env:
     APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
     APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
   shell: bash
   run: |
     CERT_PATH=$RUNNER_TEMP/certificate.p12
     KEYCHAIN_PATH=$RUNNER_TEMP/build.keychain-db
     KEYCHAIN_PASSWORD=$(openssl rand -base64 24)
     echo "$APPLE_CERTIFICATE" | base64 --decode > "$CERT_PATH"
     security create-keychain -p "$KEYCHAIN_PASSWORD" "$KEYCHAIN_PATH"
     security default-keychain -s "$KEYCHAIN_PATH"
     security set-keychain-settings -lut 21600 "$KEYCHAIN_PATH"
     security unlock-keychain -p "$KEYCHAIN_PASSWORD" "$KEYCHAIN_PATH"
     security import "$CERT_PATH" -P "$APPLE_CERTIFICATE_PASSWORD" \
       -A -t cert -f pkcs12 -k "$KEYCHAIN_PATH"
     security set-key-partition-list -S apple-tool:,apple: \
       -k "$KEYCHAIN_PASSWORD" "$KEYCHAIN_PATH"
   ```
   IMPORTANT: This step MUST precede tauri-action so `codesign` can find the identity.

7. Windows certificate import (name: "Import Windows certificate"):
   ```yaml
   if: runner.os == 'Windows' && secrets.WINDOWS_CERTIFICATE != ''
   env:
     WINDOWS_CERTIFICATE: ${{ secrets.WINDOWS_CERTIFICATE }}
     WINDOWS_CERTIFICATE_PASSWORD: ${{ secrets.WINDOWS_CERTIFICATE_PASSWORD }}
   shell: pwsh
   run: |
     New-Item -ItemType directory -Path certificate
     Set-Content -Path certificate/tempCert.txt -Value $env:WINDOWS_CERTIFICATE
     certutil -decode certificate/tempCert.txt certificate/certificate.pfx
     Remove-Item -path certificate/tempCert.txt
     Import-PfxCertificate -FilePath certificate/certificate.pfx `
       -CertStoreLocation Cert:\CurrentUser\My `
       -Password (ConvertTo-SecureString -String $env:WINDOWS_CERTIFICATE_PASSWORD `
         -Force -AsPlainText)
   ```

8. tauri-action build + release upload (name: "Build and upload release"):
   ```yaml
   uses: tauri-apps/tauri-action@v0
   env:
     GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
     # macOS signing + notarization — no-op when secrets are absent
     APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
     APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
     APPLE_SIGNING_IDENTITY: ${{ secrets.APPLE_SIGNING_IDENTITY }}
     APPLE_ID: ${{ secrets.APPLE_ID }}
     APPLE_PASSWORD: ${{ secrets.APPLE_PASSWORD }}
     APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
     # Windows signing — no-op when secrets are absent
     WINDOWS_CERTIFICATE: ${{ secrets.WINDOWS_CERTIFICATE }}
     WINDOWS_CERTIFICATE_PASSWORD: ${{ secrets.WINDOWS_CERTIFICATE_PASSWORD }}
   with:
     tagName: v__VERSION__
     releaseName: "Foliom v__VERSION__"
     releaseDraft: true
     prerelease: false
     args: ${{ matrix.args }}
   ```
   NOTE: `APPLE_PASSWORD` is the correct secret name (NOT `APPLE_ID_PASSWORD`).
   Notarization is automatic when APPLE_ID + APPLE_PASSWORD + APPLE_TEAM_ID are set.

CRITICAL correctness rules:
- The macOS keychain import step MUST use `if: runner.os == 'macOS' && secrets.APPLE_CERTIFICATE != ''`
  (note: `secrets.X != ''` is the correct GitHub Actions expression for "secret is set").
- The Windows import step MUST use `if: runner.os == 'Windows' && secrets.WINDOWS_CERTIFICATE != ''`
- When signing secrets are absent, both import steps are skipped, tauri-action runs without
  signing env vars, and the job produces unsigned artifacts and still exits 0.
- The tauri-action step passes ALL signing env vars unconditionally (using `${{ secrets.X }}`
  which evaluates to empty string when absent) — tauri-action handles the absent-secret no-op.
- Do NOT add `if:` to the tauri-action step itself.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → crates.io/npm | Dependency fetch during cargo build + npm ci |
| Signing secrets → keychain | Base64-encoded .p12/.pfx imported into ephemeral keychain; never persisted to disk beyond the job |
| tauri-action → GitHub API | GITHUB_TOKEN used for release asset upload |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Tampering | Dependency supply chain in release build | mitigate | Same mitigations as ci.yml: Cargo.lock pinned, tauri-action@v0 pinned ref, node cache uses package-lock.json hash |
| T-05-08 | Information Disclosure | Signing secrets in GitHub Actions env | mitigate | Secrets are referenced via `${{ secrets.X }}` which are masked in logs; ephemeral keychain destroyed after job |
| T-05-09 | Repudiation | Unsigned release artifacts shipped without disclosure | accept | releaseDraft: true — every release is a draft requiring human promotion; DSK-02 states "when secrets present"; build-without-signing is documented behavior per D-50-04 |
| T-05-10 | Spoofing | Tauri action tag not pinned to SHA | mitigate | `@v0` is the official Tauri-maintained tag; acceptable for an officially-maintained action. For maximum integrity, the executor may optionally pin to a specific SHA after verifying the current SHA of tauri-apps/tauri-action@v0 |
| T-05-11 | Denial of Service | Concurrent release pushes wasting runner minutes | mitigate | `concurrency: cancel-in-progress: true` cancels superseded runs on the same tag |
| T-05-SC | Tampering | cargo/npm installs in release job | mitigate | Cargo.lock + package-lock.json are committed and used via `npm ci`; Swatinem/rust-cache does not override lock |
</threat_model>
