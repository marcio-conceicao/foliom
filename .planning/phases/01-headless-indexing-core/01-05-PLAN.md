Implement `crates/core/src/scanner/walk.rs` per RESEARCH §`walkdir::filter_entry` pattern:

1. Define `pub struct ScanEntry { pub path: PathBuf, pub mtime_ns: i64, pub size: u64 }`.

2. Define `pub fn walk<'a>(root: &Path, ignore: &'a IgnoreSet) -> impl Iterator<Item = ScanEntry> + 'a`:
   - `walkdir::WalkDir::new(root).follow_links(false)` — non-negotiable for T-05-01.
   - `.into_iter().filter_entry(move |e| { ... })` — skip directories whose name is in `ignore` OR starts with `.` (dotdirs at any depth). Files are not filtered here — they survive their parent's check by virtue of descent.
   - `.filter_map(Result::ok)` — silently skip permission errors. Log via `tracing::warn!` when an error is dropped (use `inspect_err` before `filter_map(Result::ok)` to log).
   - `.filter(|e| e.file_type().is_file())`.
   - `.filter(|e| e.path().extension().and_then(|s| s.to_str()) == Some("md"))`.
   - `.filter_map(|e| { ... })` — extract metadata; on metadata failure log and skip.

3. `mtime_ns` extraction: use `std::os::unix::fs::MetadataExt::mtime` + `mtime_nsec` on Unix; on Windows use `metadata.modified()?` then convert to `SystemTime::duration_since(UNIX_EPOCH)` → nanoseconds. Wrap in a helper `fn mtime_ns_from_meta(meta: &std::fs::Metadata) -> Option<i64>` with `#[cfg(unix)]` and `#[cfg(windows)]` arms. On Unix: `meta.mtime() as i64 * 1_000_000_000 + meta.mtime_nsec() as i64`. On Windows: `meta.modified().ok()?.duration_since(UNIX_EPOCH).ok()?.as_nanos() as i64`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem → scanner | Notes folder content is user-controlled; could contain symlinks, very deep trees, files with non-UTF-8 names. |
| `config.edn` content → regex parser | User-editable file; could be malformed or hostile. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information disclosure | Symlink escape (`notes/link → /etc`) | mitigate | `walkdir::follow_links(false)` — non-negotiable. Tested with a symlink fixture on Unix. (PITFALLS §11.) |
| T-05-02 | Denial of service | Infinite directory loop via symlink cycle | mitigate | Same as above — `follow_links(false)` means walkdir does not traverse symlinks at all. |
| T-05-03 | Denial of service | Pathological directory depth (10,000 nested dirs) | accept | `walkdir` handles arbitrary depth iteratively; only memory cost is the stack of open directory handles. Reasonable corpora are well under any practical limit. |
| T-05-04 | Tampering | `config.edn` regex catastrophic backtracking | mitigate | The patterns used are linear-bounded (`[^\]\}]*` and the standard quoted-string pattern); `regex` crate guarantees linear time per the linear-time guarantee of its NFA engine. |
| T-05-05 | Information disclosure | Permissions error leaking via panic | mitigate | `filter_map(Result::ok)` swallows walkdir errors with a `tracing::warn!` log; never panics. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | New deps: `walkdir = "2.5"` (promoted from dev-dep — locked by D-17), `regex = "1"` (single-use here; standard Rust ecosystem dep). HIGH legitimacy. No `[ASSUMED]`/`[SUS]` packages. |
</threat_model>
