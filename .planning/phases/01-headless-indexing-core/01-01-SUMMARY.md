## Decisions Made

- **Corpus count assertion = 620, not 619.** Walked the corpus during test scaffolding; the real count is 620. Per A1's own guidance ("if the corpus drifts, the assertion message tells the next planner to update the count"), encoded the truth and added a comment pointing back to the assumption.
- **Reordered assertions in the round-trip test.** RESEARCH ôÏRound-Trip CI Gate puts `assert_eq!(count, 619)` before `assert!(failures.is_empty())`. With the stub (which breaks early after 3 failures), the count would only ever reach 3 in RED state, masking the actual byte-diff message under a count-mismatch panic. Swapped the order so the human-readable splice diff fires first; once plan 01-02 makes the test green both asserts pass and order is moot.
