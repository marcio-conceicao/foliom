    - `extract_refs("[[Parent/Child]]")` → PageLink "Parent/Child" (no decoding needed — the `/` is already there).

    - `extract_refs("https://example.com/page#fragment")` → empty (URL autolink suppressed by `Tag::Link` context; the `#fragment` is inside link suppression).

Implement `crates/core/src/parser/ast.rs` from RESEARCH §Stage 2. Concrete steps:

1. Define `RefKind { PageLink, Tag }` and `ExtractedRef { kind, target }` with `#[derive(Debug, Clone, PartialEq, Eq)]`.

2. `pub fn extract_refs(raw: &str) -> Vec<ExtractedRef>`: build `Parser::new_ext(raw, Options::ENABLE_TABLES | Options::ENABLE_STRIKETHROUGH | Options::ENABLE_GFM)`. Maintain `suppress_depth: u32` incremented on `Event::Start(Tag::Heading|CodeBlock|Link|Image)` and decremented on corresponding `Event::End(TagEnd::*)`. Treat `Event::Code(_)` as a no-op (never extract from inline code). Only on `Event::Text(t)` when `suppress_depth == 0`, call `scan_text_for_refs(&t)` and extend the output.

3. `fn scan_text_for_refs(text: &str) -> Vec<ExtractedRef>`: iterate by character with byte indices using `text.char_indices()`. Three patterns to recognize (in order — `#[[` is a strict prefix of `#`):
   - `[[` ... `]]` → PageLink, target = inner text after `%2F`→`/` decode and NFC normalization.
   - `#[[` ... `]]` → Tag, target same decode/normalize.
   - `#` followed by alphanumeric → bare tag; consume while char is `alphanumeric() || c == '-' || c == '_' || c == '/' || c == '.'`. Reject the captured token if: (a) it is empty, (b) it matches `^[0-9a-fA-F]{3,8}$` (hex color), (c) the byte immediately preceding `#` is `alphanumeric` (avoid URL fragments). Trim trailing `.` (since sentences end with bullets like `see #Foo.`).

4. `%2F` decode: do a literal substring replace `"%2F"` → `"/"` (case-insensitive: also handle `%2f`). Do NOT use a general percent-decoder; only this one sequence per LNK-02.

5. NFC-normalize the `target` before storing: `target.nfc().collect::<String>()`. This means `[[Avaliação]]` (NFC) and `[[Avaliac\u{0327}ão]]` (NFD) collapse to the same key — matching the way `RelativePath` normalizes filenames.

6. Module wiring: in `crates/core/src/parser/mod.rs` add `pub mod ast;` next to the existing `pub mod segment;`.

7. Doc-comment `extract_refs` with the PRS-04 reference and the suppression list. Note explicitly per AP-1: `extract_refs` is called per-block on the block's `raw` text — NEVER on the whole file. (Plan 06 enforces this by calling `extract_refs(&block.raw)` inside the indexer loop.)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User markdown text → ref extractor | Block `raw` is arbitrary user text; could contain pathologically large `[[...]]` or `#...` tokens. |
| Filesystem path → RelativePath | Filenames are user-controlled (notes folder content); could include `..`, symlink targets, non-UTF-8 bytes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of service | Block `raw` containing 10MB of `[[`...`]]` with no closing brackets | accept | Linear scan; segmenter caps block size by file size. Phase 2 may add per-block-length limit. |
| T-03-02 | Tampering | Path traversal via `..` components | mitigate | `RelativePath::from_filesystem` rejects any non-`Normal` component with `UnexpectedPathComponent`. Unit test asserts `..` is rejected. |
| T-03-03 | Information disclosure | Symlink in notes folder pointing outside root | mitigate | NOT addressed here (Phase 1 scanner sets `follow_links(false)` in Plan 05). `RelativePath::from_filesystem` does not call `canonicalize` so it cannot leak target paths. |
| T-03-04 | Tampering | Non-UTF-8 filename | mitigate | `from_filesystem` returns `NonUtf8Path` error; indexer (Plan 06) logs and skips, never panics. |
| T-03-05 | Tampering | Hex-color false-positive tags (e.g., `#deadbeef` interpreted as tag) | mitigate | Bare-tag scanner rejects `^[0-9a-fA-F]{3,8}$`. Documented edge case: `#beef` could be a real tag — `#[[beef]]` is the disambiguating syntax. Document. |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | New deps: `pulldown-cmark = "0.13"`, `unicode-normalization = "0.1"`, `thiserror = "1"`. All HIGH legitimacy (locked by CONTEXT D-10, RESEARCH §Don't Hand-Roll, D-19). No `[ASSUMED]`/`[SUS]` packages. |
