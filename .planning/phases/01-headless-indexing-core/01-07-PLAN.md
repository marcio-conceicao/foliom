## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI argv → core | User-provided arguments (paths, FTS5 queries). |
| Stdout JSON → CI/frontend | JSON is the long-lived contract between Phase 1 and Phase 2+. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | FTS5 query injection / DoS | accept | FTS5 has its own query syntax (`MATCH`); malformed queries return a SQLite error which surfaces as a clean CLI error. There is no SQL injection vector — the FTS5 query goes through a `?` parameter. Pathological FTS5 expressions could in theory be slow; for a single-user CLI this is acceptable. |
| T-07-02 | Tampering | Path traversal via `--root` argument | mitigate | `RelativePath::from_filesystem` rejects `..` components; root itself is canonicalized via `resolve_db_path`. CLI does not write outside `XDG_DATA_HOME`. |
| T-07-03 | Information disclosure | JSON output to stdout includes file paths | accept | Single-user local app; user owns stdout. Phase 5 desktop revisits if logs are shipped. |
