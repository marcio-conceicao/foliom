<objective>
Implement the pure-logic core of mutation: byte-splice into a file buffer at `(byte_offset, byte_length)`, downstream offset recomputation, and a `TreeOp` enum with invertible `apply()` semantics. NO HTTP, NO filesystem, NO SQL — all I/O lives in plans 03-01 (atomic write) and 03-03 (REST handlers).

Purpose: SNC-01 ("edits persisted by byte-splicing the changed block's bytes at `(byte_offset, byte_length)`; unchanged portions of the file remain byte-identical") is the foundational correctness invariant for Phase 3. By isolating this in a pure module, we can write exhaustive property-style tests (every fixture in `tests/fixtures/logseq-synthetic` × 5 splice patterns) without spinning up axum or sqlite.
Output: `crates/core::mutation` with `splice.rs` (`splice_block`, `compute_shifted_offsets`) and `tree_ops.rs` (`TreeOp` enum + `apply`). The HTTP layer in plan 03-03 will compose these.

```rust
#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
pub enum TreeOp {
    Indent { block_id: i64, prev_depth: u8 },
    Outdent { block_id: i64, prev_depth: u8 },
    Merge { block_id: i64, merged_into_id: i64, original_raw: String },
    Split { block_id: i64, at_offset: usize, new_block_id: i64 },
    Move { block_id: i64, prev_parent_id: Option<i64>, prev_ord: i32, new_parent_id: Option<i64>, new_ord: i32 },
    Delete { block_id: i64, snapshot: BlockSnapshot },
}

pub struct BlockSnapshot { pub raw: String, pub depth: u8, pub parent_id: Option<i64>, pub ord: i32 }

impl TreeOp {
    /// Apply to an in-memory tree representation. Returns the inverse op.
    /// Errors: TreeOpError::FirstChildCannotIndent etc.
    pub fn apply(self, tree: &mut MutableTree) -> Result<TreeOp, TreeOpError>;
}
```

`MutableTree` is a minimal in-memory representation. It is NOT a public type used elsewhere yet — it is internal to plan 03-02 / 03-03 composition. Plan 03-03 will adapt SQL rows into `MutableTree` and back.

    ```rust
    pub fn splice_block(original: &[u8], byte_offset: usize, byte_length: usize, new_raw: &[u8]) -> Vec<u8> {
        let mut out = Vec::with_capacity(original.len() - byte_length + new_raw.len());
        out.extend_from_slice(&original[..byte_offset]);
        out.extend_from_slice(new_raw);
        out.extend_from_slice(&original[byte_offset + byte_length..]);
        out
    }
    ```

    ```rust
    let delta = (new_len as i64) - (old_len as i64);
    for block in blocks {
        if block.id() == changed_block_id {
            push((id, block.byte_offset(), new_len));
        } else if block.byte_offset() > changed_block_byte_offset {
            push((id, (block.byte_offset() as i64 + delta) as usize, block.byte_length()));
        } else {
            push((id, block.byte_offset(), block.byte_length()));
        }
    }
    ```

  <behavior>
    - Building a `MutableTree` from a Vec<(id, parent_id, ord, depth, raw)> round-trips through `apply(Indent)` then `apply(inverse)` and the resulting tree equals the original (by structural equality).
    - Same for Outdent, Merge, Split, Move, Delete.
    - `Indent` on the first child of its parent (no preceding sibling at same depth) returns `Err(TreeOpError::FirstChildCannotIndent)`.
    - `Outdent` on a depth-0 block returns `Err(TreeOpError::AlreadyAtRoot)`.
    - `Merge` of a block whose predecessor is the page prelude (depth = u8::MAX) returns `Err(TreeOpError::CannotMergeIntoPrelude)`.
    - `Delete` of a block with children re-parents children to the deleted block's parent at the deleted block's `ord` position (children shift left by the deleted block's prior position).
    - Serialization round-trip: `serde_json::from_str(&serde_json::to_string(&op)?)? == op` for every variant.
  </behavior>
  <action>
    Implement `MutableTree` as a minimal in-memory representation:
    ```rust
    pub struct MutableTree {
        pub blocks: Vec<MutableBlock>,
    }
    pub struct MutableBlock {
        pub id: i64,
        pub parent_id: Option<i64>,
        pub ord: i32,
        pub depth: u8,
        pub raw: String,
    }
    impl MutableTree {
        pub fn new(blocks: Vec<MutableBlock>) -> Self;
        pub fn get(&self, id: i64) -> Option<&MutableBlock>;
        pub fn get_mut(&mut self, id: i64) -> Option<&mut MutableBlock>;
        pub fn children_of(&self, parent_id: Option<i64>) -> Vec<&MutableBlock>; // sorted by ord
        pub fn predecessor_at_same_depth(&self, id: i64) -> Option<i64>;
    }
    ```

    Implement `TreeOp::apply(self, tree: &mut MutableTree) -> Result<TreeOp, TreeOpError>` for all 6 variants per the behavior spec.

    Each variant's apply:
    - **Indent**: locate `block_id`, find `predecessor_at_same_depth` — if `None`, return `FirstChildCannotIndent`. Otherwise: set `block.parent_id = Some(predecessor.id)`, `block.depth += 1`, `block.ord = (max ord among predecessor's existing children) + 1`. Inverse: `Outdent { block_id, prev_depth: old_depth }`.
    - **Outdent**: if `block.depth == 0` (or block is direct child of prelude with depth 0), return `AlreadyAtRoot`. Otherwise: set `block.parent_id = grandparent_id`, `block.depth -= 1`, `block.ord = parent.ord + 1` (and shift later siblings of the parent right by 1). Inverse: `Indent`.
    - **Merge**: capture `merging_block.raw` into `original_raw`, append (stripped of bullet prefix) into `merged_into.raw` with a `\n` separator, remove `merging_block` from `tree.blocks`. Inverse: `Split { block_id: merging_block.id, at_offset: original merged_into.raw.len(), new_block_id: merging_block.id }`.
    - **Split**: insert a new block after `block_id` with `raw = block.raw[at_offset..]`, then truncate `block.raw = block.raw[..at_offset]`. Inverse: `Merge`.
    - **Move**: reposition block under new parent at new ord; shift siblings. Inverse: `Move` with swapped fields.
    - **Delete**: snapshot the block, re-parent its children to its parent at its ord, remove the block. Inverse: re-insert from snapshot + re-parent children back (this is the `BlockSnapshot` payload).

    Add `#[derive(serde::Serialize, serde::Deserialize)]` to `TreeOp` and `BlockSnapshot`.

    Tests in `__tests__/tree_ops_test.rs`:
    1. Build a 5-block tree: prelude (depth=255), then `A` (depth 0), `B` (depth 1, parent=A), `C` (depth 0), `D` (depth 0).
    2. For each variant, run `let inv = op.clone().apply(&mut tree)?;` then `inv.apply(&mut tree)?;` and assert the resulting tree equals the original via `#[derive(PartialEq)]` on `MutableTree`/`MutableBlock`.
    3. Edge tests: indent the first-child case (B already at depth 1 — try to indent further with no preceding depth-1 sibling), outdent-at-root (A), merge-into-prelude.
    4. Serde round-trip via `serde_json` for one op of each variant.
