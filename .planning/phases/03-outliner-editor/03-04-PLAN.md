<objective>
Stand up the CodeMirror 6 single-block editor with mount/unmount discipline, IME guard, boundary-key interception, per-instance history, and the tree-op log skeleton. Wire `Block.svelte` to swap between read-only render (Phase 2) and edit mode (this plan) on focus/click, save via blur OR Enter, and PUT to the mutation API from plan 03-03.

Purpose: This is the user-visible editor. EDT-01..07 and EDT-10 + EDT-13 all live here. Without IME guard, Pt-BR users cannot type `ã`; without `Prec.highest` boundary keymap, Enter/Tab/Backspace fire CM6 defaults instead of tree ops. Without single-instance discipline, an existing edit's IME composition is destroyed by reparenting.
Output: A user can click any block, type new markdown (including Pt-BR dead-key sequences), press Enter or blur, and see the edit persist via `PUT /api/blocks/:id`. Undo inside edit mode uses CM6's native history; undo outside edit mode pops the tree-op log (skeleton — real tree ops land in 03-05).
</objective>

    ```typescript
    export class BlockEditor {
      view: EditorView | null = null;
      mount(parent: HTMLElement, initialRaw: string, cb: BlockEditorCallbacks): void {
        if (this.view) throw new Error('BlockEditor double-mount');
        const state = EditorState.create({ doc: initialRaw, extensions: blockEditorExtensions(cb) });
        this.view = new EditorView({ state, parent });
        this.view.focus();
      }
      readDocSafe(): string | null {
        if (!this.view) return null;
        if (this.view.composing) return null;
        return this.view.state.doc.toString();
      }
      unmount(): string | null {
        if (!this.view) return null;
        const doc = this.readDocSafe();
        this.view.destroy();
        this.view = null;
        return doc;
      }
    }
    export function trySaveBlock(editor: BlockEditor): 'saved' | 'skipped-due-to-ime' | 'no-editor' {
      if (!editor.view) return 'no-editor';
      const doc = editor.readDocSafe();
      if (doc === null) return 'skipped-due-to-ime';
      // caller wires the actual PUT; this wrapper just gates IME.
      return 'saved';
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user keyboard → CM6 view → DOM | IME composition window crosses here; mishandled = corrupt user input |
| CM6 doc → `putBlock` body → backend | First write-through path; `prev_hash` carries client's known-state for conflict detection |
| 409 response → UI banner | Stale-state acknowledgement — failure to surface = silent data loss |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Repudiation | IME composition reads partial doc → corrupted `ã` → user sees `~a` saved | mitigate | `readDocSafe` returns `null` when `view.composing === true`. Tests in `ime.test.ts` pin this with happy-dom CompositionEvent dispatch. |
| T-03-11 | Tampering | Stale client save clobbers a remote-edited file | mitigate | `putBlock` includes `prev_hash`; backend 409 surfaces in UI as a Reload banner. Verified by 409 test. |
| T-03-12 | Information Disclosure | CM6 history persists across blocks (cross-block undo leaks raw of another block) | mitigate | Per 03-RESEARCH §4: each EditorView has its own EditorState → history() is per-instance → impossible to cross-undo. Test pins this with mount/unmount/remount cycle. |
| T-03-13 | DoS | Tree-op log grows unbounded → memory leak | mitigate | 200-entry cap with FIFO drop. Test in `treeOpLog.test.ts`. |
| T-03-14 | Tampering | Boundary key (Backspace at edge) handled by CM6 default before our handler | mitigate | `Prec.highest` wrap on the boundary keymap. Test (A8) in `boundary.test.ts`. |
</threat_model>
