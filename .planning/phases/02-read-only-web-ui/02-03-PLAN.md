<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /api via Vite dev proxy (dev) or rust-embed (prod) | Same-origin in both cases; no CORS surface |
| Hash route `:name` param → fetch URL | Always `encodeURIComponent(name)` before building the path segment |
| localStorage → theme store | Only `'light' | 'dark' | 'auto'` is persisted; arbitrary values are coerced to `'auto'` on load |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Tampering | URL path encoding | mitigate | `encodeURIComponent(name)` in every `api.ts` call; tested in `api.test.ts` with `Parent/Child`. |
| T-02-10 | Tampering | localStorage theme value | mitigate | On read, coerce unknown strings to `'auto'` (allowlist of three values). |
| T-02-11 | XSS via raw fetch responses | future Block renderer | accept (deferred) | Plan 02-04 introduces `{@html ...}` with `markdown-it({ html: false })`. This plan only `JSON.stringify`s into `<pre>` — no `{@html ...}` until next plan. |
| T-02-SC | Tampering | new npm deps (svelte 5, vite, svelte-spa-router, markdown-it, prismjs, vitest, happy-dom) | mitigate | Audit table in 02-RESEARCH §Package Legitimacy Audit. svelte-spa-router gated by the human-verify checkpoint above. All others [ASSUMED] — confirm publisher (Svelte, Vite, markdown-it, Prism, Vitest teams) at install. |
