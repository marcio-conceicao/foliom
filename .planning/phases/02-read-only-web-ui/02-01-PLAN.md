<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser tab ‚Üí axum on 127.0.0.1 | Untrusted JS (browser content from any origin) can attempt to reach the loopback API via DNS rebinding |
| URL path/query ‚Üí SQL | Path params and query strings become SQL bind values |
| Cargo build ‚Üí notes filesystem | Build process reads `Cargo.toml` and crate sources; no untrusted input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing / Info disclosure | axum bind | mitigate | Hardcode `SocketAddr::from(([127,0,0,1], port))`; no `--host` flag (Open Question 4). Reject Host header not in {`127.0.0.1`, `localhost`} via `host_allowlist` middleware (DNS rebinding mitigation per 02-RESEARCH ¬ßSecurity Domain). |
| T-02-02 | Tampering | startup reindex | mitigate | Reindex errors propagate via `anyhow::Context` and abort boot. Serving stale data is worse than refusing to start (D-22). |
| T-02-03 | Denial of service | tokio runtime | accept | Single-user loopback app; `current_thread` runtime is sufficient (D-25). No rate limiting needed. |
| T-02-SC | Tampering | cargo new deps (axum, tower-http, tokio, open, time) | mitigate | All [ASSUMED] in 02-RESEARCH ¬ßPackage Legitimacy Audit; verified against crates.io maintainer (Tokio team / Tower / Byron / time-rs). No new [SUS] crates introduced in this plan (svelte-spa-router gating lives in 02-03). |
</threat_model>
