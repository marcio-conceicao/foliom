# Phase 2: Read-Only Web UI — Research

**Researched:** 2026-05-21
**Domain:** Local HTTP API (axum) + Svelte 5 SPA (read-only) over Phase 1's SQLite index
**Confidence:** HIGH — stack and patterns are all locked by CONTEXT.md D-22..D-38; this document is HOW, not WHAT.

## Summary

Phase 2 extends the existing `foliom` binary with a `serve` subcommand that boots an axum 0.7 HTTP server on `127.0.0.1:7345` (configurable), backed by the Phase 1 `Db` wrapped in `Arc<Mutex<…>>`, serving a Svelte 5 + Vite SPA embedded via `rust-embed`. The frontend renders markdown per-block with `markdown-it` + three custom inline rules (`[[page]]`, `#tag`, `#[[multi word tag]]`) and Prism syntax highlighting for code fences. Navigation is hash-routed (`svelte-spa-router`); search is `Ctrl/Cmd+K` palette over FTS5 with `snippet()` highlighting; backlinks/journals/sidebar are independent REST endpoints. Two performance gates (ACPT-02 cold-start <2s, ACPT-03 RSS <300MB on 5k notes) ship as CI assertions over a synthetic generated corpus.

**Primary recommendation:** Build in 8 dependency-ordered plans. Ship the smallest possible vertical slice first (`foliom serve` + health endpoint + minimal Vite SPA + rust-embed dev/prod toggle) so the entire dev loop is exercised before any feature work lands. Treat the per-block JSON response shape and the markdown-it custom inline rules as the two highest-risk areas — they shape the entire frontend architecture and any drift here means a Phase 3 rewrite.

### Locked Decisions

**Backend (extension of Phase 1 core)**
- **D-22:** New subcommand `foliom serve <root>` on the existing binary (NO separate binary). Sequence: (a) `indexer::reindex(Incremental)` on startup, (b) open persistent `Db`, (c) axum on `127.0.0.1:<port>` (default 7345, `--port N` override), (d) print `Foliom serving at http://127.0.0.1:7345 — Ctrl+C to stop`, (e) optional browser open via `open = "5"` behind `--open` flag.
- **D-23:** **axum 0.7 + tower-http**. Frontend embedded via `rust-embed = "8"`. Dev mode delegates to Vite (`http://localhost:5173`).
- **D-24:** **REST + JSON only — no SSE/WebSocket in Phase 2.** Routes proposed (planner can adjust): `GET /api/pages`, `GET /api/pages/{name}`, `GET /api/pages/{name}/backlinks`, `GET /api/journals/today` (302), `GET /api/journals?from=&to=`, `GET /api/search?q=&limit=`, `GET /api/page-titles`.
- **D-25:** Single-threaded `tokio::current_thread`. Storage queries via `tokio::task::spawn_blocking` at handler boundary.

**Frontend**
- **D-26:** Markdown rendering on the **frontend**, not backend. Backend serves `raw` per-block; frontend renders with `markdown-it` per-block.
- **D-27:** **Svelte 5 (runes) + Vite 5 + TypeScript 5.5+**. Monorepo layout: `frontend/` at root (not under `crates/`). `frontend/dist/` embedded via `rust-embed` into `crates/cli`.
- **D-28:** **Router:** `svelte-spa-router` (hash-based: `/#/pages/Foo`, `/#/journals/2024-03-15`, `/#/search`). Block zoom is a sub-fragment: `/#/pages/Foo#block=2.1`.
- **D-29:** **markdown-it inline rules** custom for `[[page]]`, `#tag`, `#[[tag]]`. Output: `<a class="page-link" data-page="Foo">Foo</a>` / `<span class="tag" data-tag="Foo">#Foo</span>`.
- **D-30:** **Prism** (`prismjs = "1.29"`) with rust, python, js, ts, sh, sql, json, yaml, html, css statically loaded. Line numbers via `prism-line-numbers`.
- **D-31:** **Dark mode:** CSS variables + `prefers-color-scheme`, override via `localStorage('theme')`. **No CSS framework** (no Tailwind).
- **D-32:** **State:** Svelte stores built-in (`writable`, `derived`). Stores: `currentPage`, `sidebarPages`, `theme`, `searchPalette`.
- **D-33:** **Renderer per-block.** Each block receives `{ raw, properties, drawers, depth, children }` and renders via `{@html markdownIt.render(stripped_raw)}` after stripping segmenter prefix.
- **D-34:** **Block folding UI-only** in Phase 2 (no persistence — `collapsed::` property write-back is Phase 3).

**Performance gates**
- **D-35:** Synthetic 5k corpus at `crates/core/benches/fixtures/synthetic-5k/` (gitignored, generated by `crates/cli/src/bin/bench-gen.rs`). Criterion benchmark in `crates/core/benches/cold_start.rs` measures `Db::open + reindex(Full)`. ACPT-02/03 are CI assertions with **×1.5 tolerance** vs reference laptop target.
- **D-36:** **Lazy loading:** only current page's content fetched. Sidebar = name-list only (~50KB JSON for 5k notes — acceptable).

**Canonical from Phase 1**
- **D-37:** `RelativePath` for all path boundaries (URLs encode `%2F`, decode at handler).
- **D-38:** `Db` via `Arc<Mutex<Db>>` in `axum::State`. Connection pool deferred to Phase 3.

### `main` flow (concrete Rust pseudocode)

```rust
// crates/cli/src/cmd/serve/mod.rs
use anyhow::{Context, Result};
use clap::Args;
use foliom_core::indexer::{reindex, ReindexMode};
use foliom_core::storage::Db;
use std::net::{SocketAddr, TcpListener};
use std::path::PathBuf;
use std::sync::{Arc, Mutex};

#[derive(Args, Debug)]
pub struct ServeArgs {
    /// Notes root.
    pub root: PathBuf,
    /// TCP port (default 7345; pass 0 for random free port).
    #[arg(long, default_value_t = 7345)]
    pub port: u16,
    /// Open browser to the served URL.
    #[arg(long)]
    pub open: bool,
    /// Force full reindex on startup (default: incremental).
    #[arg(long)]
    pub full: bool,
}

pub fn run(args: ServeArgs) -> Result<()> {
    // 1. Reindex synchronously on startup. Failure here aborts boot —
    //    serving a stale/empty index is worse than refusing to start.
    let mut db = Db::open(&args.root).context("opening index")?;
    let mode = if args.full { ReindexMode::Full } else { ReindexMode::Incremental };
    let stats = reindex(&mut db, &args.root, mode).context("reindex on startup")?;
    tracing::info!(?stats, "reindex complete");

    // 2. Wrap DB for sharing across handlers.
    let state = state::AppState {
        db: Arc::new(Mutex::new(db)),
        root: args.root.clone(),
    };

    // 3. Bind. If port is in use, fall back to OS-assigned (port 0) and
    //    log the actual port. Never silently kill the user's other process.
    let addr: SocketAddr = ([127, 0, 0, 1], args.port).into();
    let listener = match TcpListener::bind(addr) {
        Ok(l) => l,
        Err(e) if args.port != 0 => {
            tracing::warn!("port {} in use ({}); falling back to OS-assigned", args.port, e);
            TcpListener::bind(SocketAddr::from(([127, 0, 0, 1], 0)))?
        }
        Err(e) => return Err(e.into()),
    };
    let bound = listener.local_addr()?;
    listener.set_nonblocking(true)?;

    // 4. Build runtime + router. current_thread is enough (D-25).
    let rt = tokio::runtime::Builder::new_current_thread()
        .enable_all()
        .build()?;

    let app = routes::build_router(state);

    println!("Foliom serving at http://{} — Ctrl+C to stop", bound);
    if args.open {
        browser::try_open(&format!("http://{}", bound));
    }

    rt.block_on(async move {
        let listener = tokio::net::TcpListener::from_std(listener)?;
        axum::serve(listener, app)
            .with_graceful_shutdown(shutdown_signal())
            .await?;
        Ok::<_, anyhow::Error>(())
    })
}

async fn shutdown_signal() {
    tokio::signal::ctrl_c().await.ok();
    tracing::info!("shutdown signal received");
}
```

use axum::{routing::get, Router};
use tower_http::{compression::CompressionLayer, trace::TraceLayer};

pub fn build_router(state: super::state::AppState) -> Router {
    Router::new()
        .route("/api/health", get(health::get))
        .route("/api/pages", get(pages::list))
        .route("/api/pages/:name", get(pages::detail))
        .route("/api/pages/:name/backlinks", get(pages::backlinks))

        .layer(CompressionLayer::new())

}

## REST API Schema

All responses `application/json` with `#[serde(rename_all = "camelCase")]`. Paths use `:name` (axum path param) — name is URL-encoded by the client; `%2F` namespace separator stays as `%2F` in the URL and decodes to `/` only when used as a `RelativePath` boundary value (D-37). The frontend converts `[[Parent/Child]]` → `encodeURIComponent("Parent/Child")` → `Parent%2FChild` for the path segment.

#[derive(serde::Serialize)]
#[serde(rename_all = "camelCase")]
pub struct PageDetail {
    pub name: String,
    pub is_journal: bool,
    pub formatted_title: Option<String>,   // Some(...) when is_journal, None otherwise
    pub blocks: Vec<Block>,
}

#[derive(serde::Serialize)]
#[serde(rename_all = "camelCase")]
pub struct Block {
    pub id: i64,
    pub depth: i32,                        // -1 = page prelude
    pub raw: String,                       // verbatim from blocks.raw

### Block tree shape decision (nested vs flat) — **nested wins for Phase 2**

Tradeoff:
- **Flat with `parentId`:** small JSON, frontend rebuilds tree. Best for huge pages with sparse rendering.
- **Nested `children`:** larger JSON, frontend renders recursively directly.

**Choose nested.** Justification:
- 5k-note corpus has median ~50 blocks/page; even a 1000-block page is ~50KB nested JSON.
- Recursive Svelte rendering is trivial; tree rebuild from flat array is bug-prone (cycle checks, missing parents).
- Lazy-loading (D-36) operates at the page level; we never serve all pages' blocks at once.
- Phase 3 mutations operate per-block — IDs are stable; nested shape doesn't impede splicing.

## FTS5 Query Patterns

The Phase 1 `query::search_blocks` already uses `snippet(blocks_fts, 0, '[', ']', '…', 12)`. Phase 2 changes the markers to `<mark>` / `</mark>` and increases context.

### Standard FTS query

```sql
SELECT
    p.name                                                          AS page,
    b.id                                                            AS block_id,
    snippet(blocks_fts, 0, '<mark>', '</mark>', '…', 16)            AS snippet
FROM blocks_fts
JOIN blocks b ON b.id = blocks_fts.rowid
JOIN pages  p ON p.id = b.page_id
WHERE blocks_fts MATCH ?1
ORDER BY rank          -- bm25 by default; better than b.id ASC for relevance
LIMIT ?2;
```

**`ORDER BY rank`** is FTS5's bm25 score (smaller = better; FTS5 returns negative scores). Phase 1 used `ORDER BY b.id ASC` — acceptable for CLI dump but for a search palette UX, relevance ordering matters. **Change in Phase 2.**

### Phrase queries

The user types `"speech analytics"` → frontend passes the raw query string. FTS5 supports phrase queries natively via double-quotes inside the MATCH argument:
- `"speech analytics"` → exact phrase match.
- `speech analytics` → tokens AND'd.
- `speech OR analytics` → either.

**Sanitize:** Strip backslashes, escape inner double-quotes by doubling, drop empty queries (Phase 1 already early-returns on empty). DO NOT pass user input that contains FTS5 column filters (`column:`) — block names by rejecting `:` in query strings unless wrapped in quotes (low-risk Phase 2; tighten in Phase 3 if needed).

### Tag-as-search routing

When user types `#crypto` in the palette, three paths can match:
1. **As FTS phrase:** `blocks_fts MATCH '"#crypto"'` — works if FTS tokenizer keeps `#` as token char. With `unicode61` it does NOT (it's punctuation).
2. **As refs lookup:** `SELECT b.id, p.name FROM refs r JOIN pages tp ON tp.id = r.target_page JOIN blocks b ON b.id = r.source_block JOIN pages p ON p.id = b.page_id WHERE tp.name = 'crypto' COLLATE NOCASE AND r.type = 'tag'` — this is the **correct path**.
3. **As page-name search:** `SELECT name FROM pages WHERE name LIKE '%crypto%' COLLATE NOCASE LIMIT 10`.

**Implementation:** Frontend detects leading `#` and `[[` to route the query:
- `#xxx` → call `/api/search?q=xxx&kind=tag` (handler does refs lookup).
- `[[xxx` → call `/api/page-titles` and filter client-side.
- Other → call `/api/search?q=xxx&kind=content` (FTS path).

Backend handler:

## Markdown-it Custom Inline Rules

Three rules, registered in priority order: `composite_tag` (must run before `bare_tag` so `#[[...]]` is not mis-tokenized as `#` followed by `[[...]]`), then `page_link`, then `bare_tag`. Each rule is a closure receiving `state` + `silent` and returns boolean.

### File: `frontend/src/lib/markdown/rules.ts`

```typescript
import type MarkdownIt from "markdown-it";
import type StateInline from "markdown-it/lib/rules_inline/state_inline";

const TAG_FIRST_CHAR = /[A-Za-z0-9_À-￿]/;
const TAG_CONT_CHAR  = /[A-Za-z0-9_\-\/\.À-￿]/;
const HEX_COLOR      = /^[0-9a-fA-F]{3}$|^[0-9a-fA-F]{6}$|^[0-9a-fA-F]{8}$/;

// Decode %2F for the chip's display + data-page attribute.
function canonicalize(target: string): string {
  return target.replace(/%2[Ff]/g, "/").normalize("NFC").trim();
}

// `#[[multi word tag]]` — MUST run before bare_tag.
export function compositeTag(state: StateInline, silent: boolean): boolean {
  const src = state.src;
  let pos = state.pos;
  if (src.charCodeAt(pos) !== 0x23 /* # */) return false;
  if (src.charCodeAt(pos + 1) !== 0x5B /* [ */) return false;
  if (src.charCodeAt(pos + 2) !== 0x5B /* [ */) return false;

  // URL-fragment guard: # must not be preceded by alphanumeric.
  if (pos > 0 && /[A-Za-z0-9]/.test(src[pos - 1])) return false;

  const close = src.indexOf("]]", pos + 3);
  if (close < 0) return false;

  const target = canonicalize(src.slice(pos + 3, close));
  if (!target) return false;

  if (!silent) {
    const tok = state.push("composite_tag", "span", 0);
    tok.attrSet("class", "tag composite");
    tok.attrSet("data-tag", target);
    tok.content = target;
  }
  state.pos = close + 2;
  return true;
}

// `[[Page Name]]`
export function pageLink(state: StateInline, silent: boolean): boolean {
  const src = state.src;
  const pos = state.pos;
  if (src.charCodeAt(pos) !== 0x5B) return false;
  if (src.charCodeAt(pos + 1) !== 0x5B) return false;

  const close = src.indexOf("]]", pos + 2);
  if (close < 0) return false;

  const target = canonicalize(src.slice(pos + 2, close));
  if (!target) return false;

  if (!silent) {
    const tok = state.push("page_link", "a", 0);
    tok.attrSet("class", "page-link");
    tok.attrSet("data-page", target);
    tok.attrSet("href", `#/pages/${encodeURIComponent(target)}`);
    tok.content = target;
  }
  state.pos = close + 2;
  return true;
}

// `#bare-tag`
export function bareTag(state: StateInline, silent: boolean): boolean {
  const src = state.src;
  let pos = state.pos;
  if (src.charCodeAt(pos) !== 0x23) return false;

  // URL-fragment guard.
  if (pos > 0 && /[A-Za-z0-9]/.test(src[pos - 1])) return false;

  let i = pos + 1;
  if (i >= src.length || !TAG_FIRST_CHAR.test(src[i])) return false;
  i++;
  while (i < src.length && TAG_CONT_CHAR.test(src[i])) i++;

  // Strip trailing dots (sentence terminator).
  let end = i;
  while (end > pos + 1 && src[end - 1] === ".") end--;
  if (end === pos + 1) return false;

  const token = src.slice(pos + 1, end);
  // Reject hex colors.
  if (HEX_COLOR.test(token)) return false;

  const target = canonicalize(token);
  if (!target) return false;

  if (!silent) {
    const tok = state.push("bare_tag", "span", 0);
    tok.attrSet("class", "tag");
    tok.attrSet("data-tag", target);
    tok.content = "#" + target;
  }
  state.pos = end;
  return true;
}

export function installFoliomRules(md: MarkdownIt): void {
  // Push BEFORE 'link' so `[[` doesn't get eaten by the standard link rule
  // mistakenly matching the first `[`.
  md.inline.ruler.before("link", "composite_tag", compositeTag);
  md.inline.ruler.before("link", "page_link",    pageLink);
  md.inline.ruler.before("link", "bare_tag",     bareTag);

  // Render hooks
  md.renderer.rules.composite_tag = (tokens, idx) => {
    const t = tokens[idx];
    const target = escapeHtml(t.attrGet("data-tag") || "");
    return `<span class="tag composite" data-tag="${target}">#${target}</span>`;
  };
  md.renderer.rules.page_link = (tokens, idx) => {
    const t = tokens[idx];
    const target = t.attrGet("data-page") || "";
    const href = `#/pages/${encodeURIComponent(target)}`;
    return `<a class="page-link" data-page="${escapeHtml(target)}" href="${href}">${escapeHtml(target)}</a>`;
  };
  md.renderer.rules.bare_tag = (tokens, idx) => {
    const target = escapeHtml(tokens[idx].attrGet("data-tag") || "");
    return `<span class="tag" data-tag="${target}">#${target}</span>`;
  };
}

function escapeHtml(s: string): string {
  return s.replace(/&/g, "&amp;").replace(/</g, "&lt;").replace(/>/g, "&gt;")
          .replace(/"/g, "&quot;").replace(/'/g, "&#39;");
}
```

### Segmenter-prefix stripping (frontend mirror of `ast.rs::strip_segmenter_prefix`)

```typescript
// frontend/src/lib/markdown/strip.ts
export function stripForRender(
  raw: string,
  depth: number,
  properties: Array<[string, string]>,
  drawers: Array<{ name: string; byteOffset: number; byteLength: number }>
): string {
  if (depth < 0) return ""; // prelude

  // Step 1: strip leading TABs + "- " (first line) and TABs + "  " (continuation).
  // Step 2: drop property lines (`key:: value`) — already in `properties` prop.
  // Step 3: drop drawer ranges — already represented in `drawers` prop.

  // Drawers are tricky because we receive byte offsets relative to the
  // source file, not the block. For Phase 2 we accept a simpler rule:
  // skip any line that starts with `:NAME:` and continue skipping until
  // `:END:` — mirrors the segmenter's drawer recognition.

  const lines = raw.split(/(?<=\n)/);
  const out: string[] = [];
  let inDrawer = false;

  lines.forEach((line, idx) => {
    // Strip TABs + bullet/continuation marker
    let i = 0;
    while (i < line.length && line[i] === "\t") i++;
    let body: string;
    if (idx === 0) {
      const rest = line.slice(i);
      if (rest === "-" || rest === "-\n") body = rest.endsWith("\n") ? "\n" : "";
      else if (rest.startsWith("- ")) body = rest.slice(2);
      else body = line; // non-matching — pass through
    } else {
      if (line.length >= i + 2 && line[i] === " " && line[i + 1] === " ") {
        body = line.slice(i + 2);
      } else {
        body = line;
      }
    }

    // Drawer skip
    const trimmed = body.replace(/\r?\n$/, "");
    if (inDrawer) {
      if (trimmed === ":END:") inDrawer = false;
      return; // drop drawer content
    }
    if (/^:[A-Z]+:$/.test(trimmed) && trimmed !== ":END:") {
      inDrawer = true;
      return;
    }

    // Property skip
    if (/^[A-Za-z][A-Za-z0-9._-]*::\s*/.test(trimmed)) return;

    out.push(body);
  });

  return out.join("");
}
```

- **5k notes × 50 blocks/page × keyed `{#each}`:** Svelte 5's compile-time reactivity makes recursive `<Block>` trees cheap. Mount cost for a 1000-block page is dominated by markdown-it parsing, not Svelte.
- **Budget:** 1000 blocks rendering <100ms on a reference laptop. Test in plan 02-04 with a synthetic 1000-block fixture.
- **markdown-it caching:** Don't cache rendered HTML in the store — Svelte rebuilds on prop change, and stale-cache bugs are not worth the savings on a read-only page.

use axum::{

    response::{IntoResponse, Response},

use rust_embed::RustEmbed;

#[cfg(not(debug_assertions))]
#[derive(RustEmbed)]
#[folder = "../../frontend/dist"]
struct Assets;

pub async fn serve_static(uri: Uri) -> impl IntoResponse {
    let path = uri.path().trim_start_matches('/');
    let path = if path.is_empty() { "index.html" } else { path };

    #[cfg(debug_assertions)]
    {
        // Dev: redirect to Vite (the user runs `vite dev` separately on 5173).
        // Alternative: reverse-proxy. Redirect is simpler and keeps the dev
        // experience identical to opening Vite directly.
        return Response::builder()
            .status(StatusCode::TEMPORARY_REDIRECT)
            .header(header::LOCATION, format!("http://localhost:5173/{}", path))
            .body(Body::empty())
            .unwrap();
    }

    #[cfg(not(debug_assertions))]
    {
        let asset = Assets::get(path).or_else(|| Assets::get("index.html"));
        match asset {
            Some(content) => {
                let mime = mime_guess::from_path(path).first_or_octet_stream();
                Response::builder()
                    .header(header::CONTENT_TYPE, mime.as_ref())
                    .header(header::CACHE_CONTROL, "public, max-age=3600")
                    .body(Body::from(content.data.into_owned()))
                    .unwrap()
            }
            None => StatusCode::NOT_FOUND.into_response(),
        }
    }
}

//! Generate a synthetic Logseq-style corpus for ACPT-02/03 benchmarks.
//!
//! Distribution targets a realistic real-world Logseq base:
//!   - 70% journals (one per day), 30% topical pages
//!   - File size: log-normal, median ~2KB, long tail to 50KB
//!   - Block depth: skewed to 0-2, occasional 5+
//!   - References: ~5 links/tags per block on average
//!   - 10% of pages have :LOGBOOK: drawers
//!   - 5% of blocks have `id::` or `collapsed::` properties

use clap::Parser;
use rand::{Rng, SeedableRng};
use rand_chacha::ChaCha8Rng;
use std::fs;
use std::path::PathBuf;

#[derive(Parser)]
struct Args {
    #[arg(long)] out: PathBuf,
    #[arg(long, default_value_t = 5000)] count: usize,
    #[arg(long, default_value_t = 42)] seed: u64,
}

fn main() -> anyhow::Result<()> {
    let args = Args::parse();
    let mut rng = ChaCha8Rng::seed_from_u64(args.seed);

    fs::create_dir_all(args.out.join("pages"))?;
    fs::create_dir_all(args.out.join("journals"))?;

    let n_journals = (args.count as f64 * 0.70) as usize;
    let n_pages = args.count - n_journals;

    // Generate page-name pool first so links can target real pages.
    let topical_names: Vec<String> = (0..n_pages).map(|i| format!("Topic {}", i)).collect();

    for i in 0..n_journals {
        let date = day_offset(i);                   // 2010-01-01 + i days
        let path = args.out.join("journals").join(format!("{}.md", date));
        fs::write(&path, render_journal(&mut rng, &topical_names))?;
    }
    for name in &topical_names {
        let path = args.out.join("pages").join(format!("{}.md", name));
        fs::write(&path, render_page(&mut rng, &topical_names))?;
    }
    println!("Generated {} files in {}", args.count, args.out.display());
    Ok(())
}

use foliom_core::indexer::{reindex, ReindexMode};

fn cold_start(c: &mut Criterion) {

### RSS measurement — `crates/cli/src/bin/bench-rss.rs`

```rust
//! Measure RSS after `foliom serve` boots on the 5k corpus and reaches steady state.

use std::path::PathBuf;
use std::thread;
use std::time::Duration;
use sysinfo::{Pid, System};

fn main() -> anyhow::Result<()> {
    let corpus = std::env::args().nth(1).expect("usage: bench-rss <corpus>");
    let corpus = PathBuf::from(corpus);

    // Spawn the binary as a child so we measure RSS of the production
    // build (not the bench wrapper).
    let mut child = std::process::Command::new("./target/release/foliom")
        .args(["serve", corpus.to_str().unwrap(), "--port", "7350"])
        .spawn()?;
    let pid = Pid::from_u32(child.id());

    // Wait for steady state — index complete + first page fetched.
    thread::sleep(Duration::from_secs(5));
    let _ = reqwest::blocking::get("http://127.0.0.1:7350/api/pages")?;
    thread::sleep(Duration::from_secs(2));

    let mut sys = System::new();
    sys.refresh_process(pid);
    let rss_bytes = sys.process(pid).map(|p| p.memory()).unwrap_or(0);
    let rss_mb = rss_bytes / 1024 / 1024;

    child.kill()?;
    child.wait()?;

    println!("RSS: {} MB (target: 300 MB, CI ceiling: 450 MB)", rss_mb);
    if rss_mb > 450 {
        std::process::exit(1);
    }
    Ok(())
}
```

**Note on `sysinfo` units:** `process.memory()` returns RSS in **bytes** since sysinfo 0.30. Earlier versions returned KB. Pin version explicitly to avoid silent unit drift.

## Anti-Patterns Specific to This Phase

### AP-2.1: Rendering markdown on the backend
**Trap:** Adding a Rust markdown→HTML renderer to make the backend "complete".
**Why wrong:** D-26 locks frontend rendering. Backend rendering means HTML escaping concerns, Prism on the wire (Rust syntax-highlight libs are heavier and uglier than Prism), and Phase 3 (edit) needs raw anyway — so you'd ship raw AND html in the response, doubling payload.
**Do:** Backend serves `raw` per-block; frontend renders.

### AP-2.2: Trying to write a regex-based `[[link]]`/`#tag` extractor on the frontend
**Trap:** Skipping markdown-it custom rules ("it's just regex").
**Why wrong:** Will match `[[` inside code fences, headings, URLs. Re-introduces the bug Phase 1 fixed in PRS-04.
**Do:** Use markdown-it inline rules ABOVE the standard `link` rule. Suppression inside code/headings comes from markdown-it's built-in inline-rule context.

### AP-2.3: SSE in Phase 2
**Trap:** "While we're here, let's add SSE so refresh feels live."
**Why wrong:** D-24 explicitly defers to Phase 4. SSE without a watcher is theater.
**Do:** Refresh on route change; manual reload otherwise.

### AP-2.4: SQLite connection pool
**Trap:** Reaching for `r2d2` or `deadpool-sqlite` because "concurrent handlers".
**Why wrong:** D-38. Single user, mostly-reads. `Arc<Mutex<Db>>` + `spawn_blocking` is correct and simpler.
**Do:** Pool only if Phase 3 surfaces contention. Don't preemptively pool.

### AP-2.5: SvelteKit instead of Svelte + Vite
**Trap:** "SvelteKit is the official meta-framework."
**Why wrong:** SvelteKit's file-system routing, server endpoints, and SSR adapter are misaligned with the single-binary embedded-via-rust-embed model. You'd fight the framework.
**Do:** Plain Svelte 5 + Vite, `svelte-spa-router` for routes.

### AP-2.6: Tailwind / utility-first CSS
**Trap:** "Faster styling."
**Why wrong:** D-31 explicit ban. Tailwind adds ~30KB even with purge, fights with theming via CSS variables, and the app has ~6 components — no scale to amortize against.
**Do:** Vanilla CSS + custom properties for theming. Per-component `<style>` blocks (Svelte scopes them).

### AP-2.7: HTTP polling for "live feel"
**Trap:** `setInterval(() => refetch(), 1000)` to mimic SSE.
**Why wrong:** Murders the perf budget (RSS grows with FTS5 mmap, SQLite cache warms over time). Read-only Phase 2 has no source of change that the user didn't trigger — no need for polling.
**Do:** Refresh on user action (route change, search). Done.

### AP-2.8: Caching rendered HTML in a Svelte store
**Trap:** "markdown-it is slow on big pages; cache the HTML."
**Why wrong:** Stale-cache bugs proliferate; D-33 says rendering is per-block (already cheap). Phase 3 edit invalidation becomes another bug surface.
**Do:** Render on demand. Profile first — if a single page renders <100ms, no cache needed.

## Common Pitfalls

### Pitfall 1: Path encoding round-trip breakage
**What goes wrong:** Frontend builds `#/pages/Parent/Child` (slash) → svelte-spa-router routes to `Parent` with `Child` as remainder, OR encodes incorrectly.
**Why:** Forward slashes in URL path segments are reserved.
**How to avoid:** Always `encodeURIComponent(name)` before building the route. The encoded form is `Parent%2FChild`. svelte-spa-router decodes once when handing the param to your component — `:name` in the route pattern matches `Parent%2FChild` and component receives `"Parent/Child"`. **Test fixture:** `Parent%2FChild.md` is committed (`crates/core/tests/fixtures/logseq-synthetic/pages/Parent%2FChild.md`) — automated test covers the full round-trip.

### Pitfall 2: Tab/heading collision in markdown-it
**What goes wrong:** ATX heading `### #Bruno` produces a `<h3>` containing a `#Bruno` tag chip — chip should NOT be extracted (PRS-04 invariant).
**Why:** markdown-it runs inline rules inside heading text by default.
**How to avoid:** See the heading-skip decision in §Markdown-it Custom Rules. Lock the chosen mechanism in plan 02-04. **Test:** fixture `06-hex-url-heading-not-tag.md` already covers this — mirror its expected output in vitest tests against frontend rendering.

### Pitfall 3: Segmenter prefix leaks into rendered output
**What goes wrong:** Block raw `\t- Foo\n` rendered verbatim by markdown-it produces `<pre><code>Foo</code></pre>` (TAB = indented code block).
**Why:** CommonMark sees TAB as 4-space indent.
**How to avoid:** Always run `stripForRender` before passing to `md.render`. **Test:** every block in every fixture round-trips through the frontend `stripForRender` + `md.render` with expected HTML.

### Pitfall 4: rust-embed empty dist crash
**What goes wrong:** `cargo build` fails because `frontend/dist` doesn't exist on a fresh clone.
**How to avoid:** Commit `frontend/dist/.gitkeep`. Document "run `cd frontend && npm install && npm run build` first" in the README.

### Pitfall 5: Windows curl in CI smoke test
**What goes wrong:** GitHub Actions Windows runners have curl but its behavior diverges from Linux curl (Powershell aliases `curl` to `Invoke-WebRequest`).
**How to avoid:** Use `Invoke-RestMethod` explicitly in the PowerShell smoke step (already in §CI Matrix Updates).

### Pitfall 6: FTS5 snippet truncates mid-multibyte
**What goes wrong:** Portuguese text "Avaliação" snipped at the `ç` byte boundary → invalid UTF-8 surfacing as `�` replacement.
**How to avoid:** FTS5 `snippet()` is unicode-aware in `unicode61` tokenizer mode (which is what we use). Safe. **Verify** in plan 02-02 tests with a Portuguese fixture.

### Pitfall 7: Time zone in `/api/journals/today`
**What goes wrong:** Server local time vs user local time disagree (server in UTC, user in São Paulo).
**How to avoid:** Use `time::OffsetDateTime::now_local()` (which reads `TZ` env). Document: "today" follows the machine's local time. For a single-user local app, server time = user time always. **Test:** mock `now()` in tests.

### Pitfall 8: Concurrent reindex on save (Phase 3 forward-leak)
**What goes wrong:** Phase 2 ships with `Arc<Mutex<Db>>` and a single startup reindex. Phase 3 will add an edit endpoint that triggers per-file reindex. If we serialize too aggressively now, Phase 3 hits contention.
**How to avoid:** Wrap every DB access in `spawn_blocking(move || { let db = state.db.lock().unwrap(); ... })`. Releases the tokio task while waiting. Phase 3 layers connection pooling on top if needed (D-38).

### Pitfall 9: Vite dev proxy CORS confusion
**What goes wrong:** `fetch('/api/...')` in Vite dev hits Vite's proxy → 7345 axum. If axum sends `Access-Control-Allow-Origin: *` and frontend sends credentials, browsers reject.
**How to avoid:** Don't add CORS at all in Phase 2. Same-origin (Vite proxy makes the browser see all requests as same-origin to 5173). Add `tower-http::cors` only if browsers complain. For a local app: never needed.

### Pitfall 10: Bundle bloat from Prism langs
**What goes wrong:** Importing all Prism languages = 200KB+.
**How to avoid:** Static imports of the ~10 langs in D-30 only. Use the `prismjs/components/prism-XXX` paths. **CI gate:** bundle size budget check (§CI Matrix Updates).

### axum handler — backlinks (file: `crates/cli/src/cmd/serve/routes/pages.rs`)

```rust
use axum::{extract::{Path, State}, Json, http::StatusCode};
use serde::Serialize;
use tokio::task;
use crate::cmd::serve::state::AppState;

#[derive(Serialize)]
#[serde(rename_all = "camelCase")]
pub struct Backlink {
    pub page: String,
    pub block_id: i64,
    pub snippet: String,
}

pub async fn backlinks(
    Path(name): Path<String>,
    State(state): State<AppState>,
) -> Result<Json<Vec<Backlink>>, StatusCode> {
    // Path param arrives URL-decoded by axum — `Parent%2FChild` → `Parent/Child`.
    // That matches our canonical page name (D-37).
    let result = task::spawn_blocking(move || {
        let db = state.db.lock().expect("poisoned");
        let conn = db.conn();
        let mut stmt = conn.prepare(
            r#"
            SELECT p.name, b.id, substr(b.raw, 1, 200)
            FROM refs r
            JOIN blocks b ON b.id = r.source_block
            JOIN pages  p ON p.id = b.page_id
            JOIN pages tp ON tp.id = r.target_page
            WHERE tp.name = ?1 COLLATE NOCASE
            ORDER BY p.name, b.ord
            LIMIT 500
            "#,
        ).map_err(|_| StatusCode::INTERNAL_SERVER_ERROR)?;
        let rows = stmt
            .query_map([name], |r| {
                Ok(Backlink {
                    page: r.get(0)?,
                    block_id: r.get(1)?,
                    snippet: r.get(2)?,
                })
            })
            .map_err(|_| StatusCode::INTERNAL_SERVER_ERROR)?
            .collect::<Result<Vec<_>, _>>()
            .map_err(|_| StatusCode::INTERNAL_SERVER_ERROR)?;
        Ok::<_, StatusCode>(rows)
    })
    .await
    .map_err(|_| StatusCode::INTERNAL_SERVER_ERROR)??;

    Ok(Json(result))
}
```

### Journal date formatter (file: `crates/cli/src/cmd/serve/format.rs`)

use time::{Date, Month};

/// "May 21st, 2026" — English long form with ordinal.
pub fn format_journal_title(date: Date) -> String {
    let day = date.day();
    let suffix = ordinal_suffix(day);
    let month = month_name(date.month());
    let year = date.year();
    format!("{} {}{}, {}", month, day, suffix, year)
}

fn ordinal_suffix(day: u8) -> &'static str {
    match day {
        11 | 12 | 13 => "th",
        n if n % 10 == 1 => "st",
        n if n % 10 == 2 => "nd",
        n if n % 10 == 3 => "rd",
        _ => "th",
    }
}

fn month_name(m: Month) -> &'static str {
    match m {
        Month::January => "January", Month::February => "February",
        Month::March => "March", Month::April => "April",
        Month::May => "May", Month::June => "June",
        Month::July => "July", Month::August => "August",
        Month::September => "September", Month::October => "October",
        Month::November => "November", Month::December => "December",
    }
}

/// Parse `YYYY_MM_DD` (journal filename stem) → Date.
pub fn parse_journal_name(name: &str) -> Option<Date> {
    let parts: Vec<&str> = name.split('_').collect();
    if parts.len() != 3 { return None; }
    let y: i32 = parts[0].parse().ok()?;
    let m: u8 = parts[1].parse().ok()?;
    let d: u8 = parts[2].parse().ok()?;
    let month = Month::try_from(m).ok()?;
    Date::from_calendar_date(y, month, d).ok()
}

## Security Domain

`security_enforcement` default = enabled.

### Applicable ASVS Categories

| ASVS Category | Applies | Standard Control |
|---------------|---------|-----------------|
| V2 Authentication | no | Single-user local app; loopback only |
| V3 Session Management | no | No sessions in Phase 2 |
| V4 Access Control | yes | Bind `127.0.0.1` only (D-22); reject non-localhost `Origin` (defer to plan 02-01) |
| V5 Input Validation | yes | URL path param `:name` — sanitize before SQL bind (parameterized via rusqlite `params![]`); FTS5 MATCH input sanitization (§FTS5 Query Patterns) |
| V6 Cryptography | no | None in Phase 2 |
| V14 Data Protection | yes | Never log file contents (Phase 1 invariant) |

### Known Threat Patterns for {axum + browser SPA on loopback}

| Pattern | STRIDE | Standard Mitigation |
|---------|--------|---------------------|
| DNS rebinding (attacker site → reach loopback API) | Tampering / Info disclosure | Reject requests where `Host` header is not `127.0.0.1:<port>` or `localhost:<port>`. tower-http does not enforce this by default — add a small middleware in plan 02-01. |
| SQL injection via path/query params | Tampering | Always use `rusqlite::params![]`; never string-format SQL |
| XSS via raw markdown `<script>` | Tampering | `markdown-it({ html: false })` (default) — explicitly set; never use `{@html userInput}` outside `md.render()` |
| FTS5 query column injection | Tampering | Reject `:` in user query (or wrap whole query in double-quotes) |
| Symlink traversal from notes root | Info disclosure | Phase 1 `RelativePath::from_filesystem` already rejects non-`Component::Normal` — same boundary enforced here |
| Path traversal in `:name` param | Info disclosure | `:name` is matched against `pages.name` (canonical) — no filesystem path concatenation; the name is purely a SQL bind value |
| LAN exposure if bound to `0.0.0.0` | Info disclosure | Hardcode `127.0.0.1` in `SocketAddr` (no `--host` flag) |

## Open Questions

1. **ATX heading suppression mechanism for markdown-it inline rules.**
   - What we know: markdown-it runs inline rules inside headings; we don't want chips inside headings (PRS-04 invariant).
   - What's unclear: Cleanest implementation — env flag, separate md instance, or token post-processing.
   - Recommendation: Plan 02-04 first task is a 30-min spike trying all three; pick simplest.

2. **Block subtree zoom vs scroll-into-view for LNK-07.**
   - What we know: URL fragment `#block=N` is the contract.
   - What's unclear: Whether Phase 2 ships subtree-only rendering (heavier UX) or just scroll-and-highlight (lighter).
   - Recommendation: Ship scroll-and-highlight in Phase 2; subtree zoom is a stretch goal. Document the limit.

3. **Search palette behavior on empty result.**
   - What we know: SCH-01..03 specify search exists.
   - What's unclear: Show "No results" + suggest creating a page? Or silent empty list?
   - Recommendation: "No results matching 'xxx'. Press Enter to search differently." No create-page action in Phase 2 (that's Phase 3 LNK-04).

4. **Whether to expose `--host` flag.**
   - What we know: D-22 bind 127.0.0.1. No mention of overriding.
   - What's unclear: Future Tauri integration may need different bind semantics.
   - Recommendation: Don't expose `--host` in Phase 2. Phase 5 (Tauri) revisits.
