## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL path/query â†’ SQL bind | All path/query params become rusqlite `params![]` bind values; never string-interpolated into SQL |
| URL %2F â†’ canonical page name | axum decodes `Parent%2FChild` to `Parent/Child` at the path extractor; SQL matches against `pages.name COLLATE NOCASE` |
| Search query â†’ FTS5 MATCH | User text becomes the right-hand side of `MATCH`; column filters (`col:`) and backslashes are rejected/stripped |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | All handlers | mitigate | rusqlite `params![]` for every bind; zero string-formatted SQL (audit gate in code review). |
| T-02-05 | Tampering | search handler | mitigate | Strip backslashes, reject `:` in unquoted query, double inner `"`. Backslash strip is conservative â€” accept that some legitimate queries (`\d`) won't match. |
| T-02-06 | Information disclosure | path traversal via `:name` | mitigate | `:name` is a SQL bind value only; never concatenated into a filesystem path. Phase 1's `RelativePath::from_filesystem` boundary stays the only fs gate. |
| T-02-07 | Information disclosure | error logging | mitigate | Map all DB errors to `StatusCode::INTERNAL_SERVER_ERROR`; log via `tracing::error!` with the error's `Display`, never file contents (V14). |
| T-02-08 | Repudiation | n/a (no auth) | accept | Single-user local app â€” no audit log requirement. |
