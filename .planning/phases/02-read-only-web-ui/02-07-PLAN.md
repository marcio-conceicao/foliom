<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Compile-time bundling of `frontend/dist` → binary | Trust scope is the dev machine (whatever produced the bundle gets embedded). No runtime input crosses here. |
| URL path → rust-embed key lookup | Path-traversal not possible because rust-embed's `Assets::get(path)` only matches against the embedded asset map; `..` segments never resolve to outside the dist folder. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-23 | Tampering | embedded assets | accept | Single-user local app; the binary is built from local source; integrity is the developer's machine integrity. |
| T-02-24 | Info disclosure | dev-mode redirect to localhost:5173 | accept | Dev-only path; never reaches production binary because `cfg(debug_assertions)` guards it. |
| T-02-SC | Tampering | new crates (rust-embed, mime_guess) | mitigate | Both [ASSUMED] in 02-RESEARCH audit; both have wide adoption (pyrossh/rust-embed, abonander/mime_guess). Pin major. |
</threat_model>
